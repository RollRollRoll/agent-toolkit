# Refine Idea

## 用途

在动手写 spec 或代码之前，把一个还没想透的想法 / 期望，通过协作对话
打磨成一个**边界清晰的可执行概念**：锁定"做什么"和"不做什么"，
但刻意不产出技术方案、不拆解任务。

它填补"挖意图"和"写 spec"之间的空档：止步于概念边界，把确认过的
概念交还用户，不自动往下进入实现。

## 触发场景

- "我想做一个 X（但还没想清楚）/ 我有个想法，帮我理一理"
- "帮我把这个需求打磨打磨 / 这个点子靠谱吗"
- "在动手前先把要做什么、不做什么定下来"
- 需求宽泛、模糊，或夹带 "scalable / 干净的架构 / 业界标准做法"
  这类"听上去应该想要"的词，需要先厘清真实意图。
- 不适用：需求已明确可直接写 spec / 编码；要的是技术选型 / 架构 /
  实现方案；已在写代码 / 调 bug / 评审；以及任何非交互场景
  （CI / 定时任务 / 自治循环）。

## 使用方式

将本目录下的 `SKILL.md` 和 `references/` 复制到目标平台的 skill 目录
（Claude Code：`.claude/skills/refine-idea/`；Codex：`.agents/skills/refine-idea/`）即可直接使用。

## 目录说明

- `SKILL.md`：skill 主体（平台原生格式，含 frontmatter）。
- `references/lenses.md`：阶段 2「照亮边界」用的思维透镜清单。
- `references/concept-note.md`：阶段 4 落盘时用的概念单模板。
- `docs/`：开发过程中的设计文档。
