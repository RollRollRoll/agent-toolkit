# 概念单模板 — 落盘用

> 仅在用户**明确要求落盘**时，按本模板写到 `docs/ideas/<名字>.md`
> （或用户指定位置；目录不存在先跟用户确认）。
>
> **铁律**：概念单只写到"概念边界"为止——**绝不含**技术选型、架构、
> 数据模型、任务拆解、排期。那些是下游 spec / plan 的事。
>
> **保持轻量**：整份一页以内，每节几行。空着的节宁可写"暂无 / 待定"，
> 不要硬填。

## 模板

```markdown
# 概念：<名字>

## 要解决的问题
<一句话写清真实意图——是什么痛点，不是"要做什么东西">

## 给谁 / 为什么是现在
<目标用户；以及为什么这个问题现在值得解决>

## 做什么
- <功能 / 产品边界**之内**的事，逐条列>

## 不做什么 ⭐
- <显式的范围外。这一节是整份文档最有价值的部分，不能空>

## 成功长什么样
<事后能拿来对照、可判断的标准。避免"能用就行"这类弱标准>

## 待验证的关键假设
- <假设> —— 验证方法：<怎么用最小代价证实 / 证伪>

## 悬而未决的问题
- <还没定、但不阻塞"概念成立"的点>
```

## 写完自查

- [ ] "不做什么"非空，且不是泛泛而谈？
- [ ] 通篇没有技术选型 / 架构 / 任务拆解？
- [ ] 每条关键假设都附了"怎么验证"？
- [ ] 成功标准是可判断的，不是"能用就行"？
