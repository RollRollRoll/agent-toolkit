# 任务清单模板 — 落盘用

> 用途：`split-task` 阶段 2 起草、阶段 4 落盘时用本模板。
> 默认写到仓库既有 tasks 位置；无既有约定时默认 `docs/tasks/<YYYY-MM-DD>-<主题>-tasks.md`。
>
> **铁律**：只到**任务拆分层**——每个任务可独立验收，但不写逐行实现代码 / 含代码施工单（那是执行）。
> 每个任务都要有"验收标准 + 验证方式"；design 每个组件、spec 每条 MUST / MUST NOT 都要有任务落点。
>
> **brownfield 用 delta**：只列改动相关任务 + 迁移 / 回滚，不重列与本次无关的既有工作。
> 迁移 / 回滚只要判定为“需任务”，也必须使用完整的 `### Task N:` 结构，不能只写游离 bullet。

## 模板（greenfield / 全量）

```markdown
# 任务清单：<主题>

## 背景 / 范围
- 实现哪份 design / spec（链接 / 路径）。
- 本次覆盖什么；显式不覆盖什么。

## 任务列表

### Task 1: <一句话目标>
- 切片：<属于哪条垂直切片 / 或：地基任务>
- 涉及文件：<创建 / 修改 / 测试的文件>
- 依赖：<前置任务，无则 none>；可并行：<是 / 否>；高风险：<是 / 否>
- 验收标准：<完成判据，可被另一个人独立判断>
- 验证方式：<怎么验证，最好对应"测试 / 构建怎么跑"，如 `pytest tests/test_x.py`>
- 覆盖：<落地的 design 组件 / spec 条目>
- [ ] 完成

### Task 2: <一句话目标>
（同构）

## 依赖与并行视图
- 顺序：<拓扑顺序，如 T1 → T2 → T3>
- 可并行：<哪些任务可并行且不冲突文件，如 T2 ∥ T3；无则注明线性>
- checkpoint：<在哪几步设检查点、检查什么>

## 覆盖核对表
| design / spec 条目 | 任务落点 |
|---|---|
| <design 组件 / Decision> | <Task N> |
| <spec Requirement> | <Task N 的验收> |
| <spec MUST NOT> | <Task N 收口 / 验收项> |

## 待解问题 / 延后
- <延后到后续版本、或还没定的点>
```

## 模板（brownfield / delta）

```markdown
# 任务清单（delta）：<这次改什么>

## 背景 / 范围
- 实现哪份 design（delta）/ spec；本次改动范围；非目标。

## 任务列表（只列改动相关）

### Task 1: <目标>
- 切片 / 涉及文件 / 依赖 / 验收标准 / 验证方式 / 覆盖（同 greenfield）
- 对现有的影响：<动了哪些现有行为 / 文件，怎么不破坏>
- [ ] 完成

### Task 2: <迁移 / 兼容目标，仅在探针判定“需任务”时保留>
- 切片：<迁移 / 兼容>
- 涉及文件：<迁移脚本、兼容层、测试等>
- 依赖：<前置任务>；可并行：<是 / 否>；高风险：<是 / 否>
- 验收标准：<迁移后与兼容期的可验证结果>
- 验证方式：<前向迁移、兼容性测试等>
- 覆盖：<design / spec 条目>
- 对现有的影响：<现有数据 / 行为如何保持正确>
- [ ] 完成

### Task 3: <回滚 / 降级目标，仅在探针判定“需任务”时保留>
- 切片：<回滚 / 降级>
- 涉及文件：<回滚脚本、feature flag、runbook、测试等>
- 依赖：<前置任务>；可并行：<是 / 否>；高风险：<是 / 否>
- 验收标准：<回滚 / 降级后的可验证结果>
- 验证方式：<回滚演练、降级测试等>
- 覆盖：<design / spec 条目>
- 对现有的影响：<失败时如何恢复>
- [ ] 完成

## 迁移 / 回滚索引（只做汇总，不代替 Task）
- 迁移 / 兼容：<Task N；或“不适用：原因”>
- 回滚 / 降级：<Task N；或“不适用：原因”>

## 依赖与并行视图 / 覆盖核对表
（同 greenfield）

## 待解问题 / 延后
- <延后或还没定的点>
```

## 写完自查

- [ ] 每个任务都有**验收标准 + 验证方式**，可被独立判断？
- [ ] 每个**拆分探针维度**都表态了（需任务 / 不适用 / 延后）？
- [ ] design 每个组件、spec 每条 Requirement / MUST NOT 都在**覆盖核对表**有落点？没有镀金任务？
- [ ] 依赖关系**无环**、顺序可执行，高风险前置、设了 checkpoint？
- [ ] 任务大小 S / M（无 XL 没拆、无过碎该并），垂直切片优先？
- [ ] 通篇**没下沉到逐行实现代码 / 含代码施工单**？
- [ ] brownfield 用了 delta，列了**迁移 / 回滚**与对现有影响，没重列全量？
- [ ] 所有“需任务”的迁移 / 兼容 / 回滚都用了完整 `Task N`，没有游离 bullet？
- [ ] 推断出来的拆分都标注了**假设**？
