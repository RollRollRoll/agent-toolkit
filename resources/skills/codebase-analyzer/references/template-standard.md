# 项目调研报告模板 · 标准调研档

> **这是「标准调研」档的骨架**，对应 SKILL "输出深度"里的**默认档**——目标是让读者建立**完整心智模型**，但不堆砌、不为凑满章节硬撑。
> 适用：用户没特别说明深浅时的默认选择。需要更轻 → [template-quick.md](template-quick.md)；需要交付级完整报告 → [template-deep.md](template-deep.md)。
>
> **原则**：
> - 抓主干，覆盖下面每个章节；与项目无关的点**一行带过**，不硬撑子标题。
> - 关键结论给**源码路径**，不准凭名字 / README 猜；没查清的写"未搞清楚（原因）"，确实没有的写"不适用（原因）"，两者别混。
> - 端到端流程是信息密度最高的部分，**不要省**。
