# Codebase Analyzer 报告内目录（TOC）：标准 / 深度档加可跳转章节目录

> **日期**：2026-06-06
> **状态**：设计待确认
> **产出方式**：brainstorming 协作
> **关联**：继 [2026-06-04-codebase-analyzer-output-destination-design.md](2026-06-04-codebase-analyzer-output-destination-design.md) 之后的又一次报告呈现增强；沿用"路线 A：扩展现有文件、不新建"

---

## 1. 背景与目标

### 1.1 痛点

现有调研报告（标准档 9 章、深度档 10 章 + 附录）正文**没有章节目录**。深度报告尤其长，读者打开后：

- **缺全局结构感**：不知道报告有哪些章节、有多长。
- **导航成本高**：想跳到某一章（如"端到端流程""综合评价"）只能手动滚动 / 搜索。

### 1.2 目标

在**标准档、深度档**报告正文里增加一个**可点击跳转的章节目录（TOC）**，放在报告最前（一页纸概览之前），让读者一眼掌握结构、并能点击跳转到任一一级章节。

### 1.3 关键权衡（明示）

- **锚点可靠性是核心风险**：报告是 Claude 生成的文本，中文标题（尤其带 `.`、`/`、`（）`）的 Markdown 锚点跨渲染器不一致、**动态生成易写错**，坏链接比没链接更糟。
  - **解法**：目录只列一级章节，而一级章节标题在模板里是**固定文本**（无 `{{占位}}`）。因此目录段可**预置进模板、锚点一次性写死**，Claude 产报告时**照抄即可、不动态生成** —— 从根上消除写错风险。
- **渲染器差异（残余取舍）**：锚点按 GitHub GFM 规则预置；GitHub 与多数编辑器一致，个别渲染器对多标点标题（如 `9. 未搞清楚 / 假设`）的锚点算法可能差一两个字符。这是选择"原生锚点链接"形态的固有取舍，预置已将其最小化。
- **快速档不加**：快速概览仅半屏，目录会喧宾夺主、比正文还抢眼，故只在标准 / 深度档加。

---

## 2. 核心设计决策

| 决策 | 结论 |
|---|---|
| 落地路线 | 扩展现有文件，不新建（沿用路线 A） |
| 适用档位 | **标准 + 深度**加；快速概览不加 |
| 目录形态 | 原生 Markdown 锚点链接 `[标题](#锚点)`，可点击跳转 |
| 层级深度 | 只列一级 `##` 章节（标准 10 条 / 深度 12 条） |
| 摆放位置 | 「一页纸概览」**之前**，紧跟标题 / 元信息 |
| 显示文字 | = 章节 `##` 标题原文（与锚点同源，最可靠、好维护） |
| 可靠性策略 | 目录段**预置进模板、锚点写死**，Claude 照抄不自算 |

**为什么显示文字用标题原文**：目录项是 `[显示文字](#锚点)`。若显示文字简化、锚点却要按真实标题算，Claude（或维护者）需同时维护两个不同字符串，易出现"显示与锚点对不上 → 跳不到"。让显示文字 = 标题原文，则显示与锚点**同源**，改标题时一处同步，最不易错。代价是深度档少数条目偏长（如 `9. 综合评价（亮点 / 适合度 / 二次开发）`），可接受。

## 3. 目录段定稿（预置内容）

### 3.1 锚点生成规则（GitHub GFM）

以下规则**以注释形式保留在模板里**，作为维护指引（日常 Claude 照抄即可、不需执行；仅在未来增删 / 改名章节时据此重算锚点）：

1. 取 `##` 标题原文（去掉 `## ` 前缀）。
2. 转小写（中文不变；英文字母如 `A`→`a`）。
3. 删除标点：`.`、`/`、`（`、`）`、`：`、`、` 等（保留中文、字母、数字、空格、连字符）。
4. 空格 → 连字符 `-`（**连续空格 → 连续连字符**，例如 `/` 两侧各有空格时产生 `--`）。

**worked examples**（用模板里真实的硬标题）：

- `## 6. 配置、依赖与运行` → `#6-配置依赖与运行`
- `## 9. 综合评价（亮点 / 适合度 / 二次开发）` → `#9-综合评价亮点--适合度--二次开发`
- `## 10. 未搞清楚的部分 / 假设 / 已知问题` → `#10-未搞清楚的部分--假设--已知问题`
- `## 附录 A：调研路径回放（可选）` → `#附录-a调研路径回放可选`

### 3.2 标准档目录段（10 条）

插入位置：`references/template-standard.md` 的 `## 一页纸概览` 之前。

```markdown
## 目录

- [一页纸概览](#一页纸概览)
- [1. 项目定位与功能](#1-项目定位与功能)
- [2. 整体架构](#2-整体架构)
- [3. 技术栈](#3-技术栈)
- [4. 代码结构与核心实现](#4-代码结构与核心实现)
- [5. 端到端代表性流程](#5-端到端代表性流程)
- [6. 配置、依赖与运行](#6-配置依赖与运行)
- [7. 项目类型专项分析](#7-项目类型专项分析)
- [8. 综合评价](#8-综合评价)
- [9. 未搞清楚 / 假设 / 已知问题](#9-未搞清楚--假设--已知问题)
```

### 3.3 深度档目录段（12 条）

插入位置：`references/template-deep.md` 的 `## 一页纸概览` 之前。

```markdown
## 目录

- [一页纸概览](#一页纸概览)
- [1. 项目定位与功能](#1-项目定位与功能)
- [2. 整体架构](#2-整体架构)
- [3. 技术栈](#3-技术栈)
- [4. 代码结构与核心实现](#4-代码结构与核心实现)
- [5. 端到端代表性流程](#5-端到端代表性流程)
- [6. 配置、依赖与运行](#6-配置依赖与运行)
- [7. 工程实践与质量](#7-工程实践与质量)
- [8. 项目类型专项分析](#8-项目类型专项分析)
- [9. 综合评价（亮点 / 适合度 / 二次开发）](#9-综合评价亮点--适合度--二次开发)
- [10. 未搞清楚的部分 / 假设 / 已知问题](#10-未搞清楚的部分--假设--已知问题)
- [附录 A：调研路径回放（可选）](#附录-a调研路径回放可选)
```

> **分隔风格**：目录段前后各加一条 `---`，与模板现有"标题 → 元信息 → `---` → 一页纸概览"的分隔风格一致。最终顺序为：`# 报告标题` → `> 元信息` → `---` → `## 目录` → `---` → `## 一页纸概览` → …

## 4. 落地文件改动清单

### 4.1 `references/template-standard.md`（核心）

在 `## 一页纸概览` 前插入 §3.2 的 10 条目录段（含前后 `---`）。

### 4.2 `references/template-deep.md`（核心）

- 在 `## 一页纸概览` 前插入 §3.3 的 12 条目录段（含前后 `---`）。
- **附录 A 标题始终保留**：深度档本就要求"每个章节都保留标题、不省略"，目录含附录 A 链接不会产生死链。可选的是附录**内容**（没内容写"不适用 / 未启用"），不是**标题**。

### 4.3 `SKILL.md`（核心）

第 5 步自检「🟡 标准调研档」新增一条（深度档作为标准档超集自动继承）：

> `- [ ] 报告开头（一页纸概览前）是否放了「## 目录」章节目录？其条目与各 ## 章节真实标题一一对应、可点击跳转？`

> 措辞注意：用"**章节目录** / 「## 目录」"指代 TOC，与第 4 节已有的"**目录地图**"（项目目录树）区分，避免混淆。

### 4.4 `references/parallel-strategy.md`（极小 / 可选）

大项目深度档"按报告章节并行起草"时，目录段是模板固定内容、不依赖子 agent，由**主 agent 在最终聚合版**照模板加在概览前即可。因影响极小，至多在「按报告章节并行起草」的聚合步骤点一句；若一句都不加亦不影响正确性（模板已自带目录段）。**倾向不改或仅补半句**，保持精准修改。

## 5. 验收标准

1. `template-standard.md` 在 `## 一页纸概览` 前有 10 条目录，锚点与"一页纸概览 + 1~9 章"标题一一对应。
2. `template-deep.md` 在 `## 一页纸概览` 前有 12 条目录，含附录 A，锚点与"一页纸概览 + 1~10 章 + 附录 A"标题一一对应。
3. `template-quick.md`（快速概览档）**不加**目录。
4. 目录只到一级 `##`，不含二级小节（1.1 / 1.2 …）。
5. 目录显示文字 = 各章节 `##` 标题原文；锚点按 §3.1 GitHub 规则。
6. `SKILL.md` 第 5 步自检「标准调研档」已加"章节目录核对"条，且与"目录地图"用词区分。
7. 任何章节的 `##` 标题文字**未被改动**（本次只新增目录段）。
8. 改完模板后同步到 `~/.claude/skills`（开发仓库与安装副本是两份独立拷贝）。

---

## 6. 不在本次范围（Non-goals）

- 快速概览档不加目录。
- 不列到二级小节（`###`）。
- 不改任何章节 `##` 标题文字。
- 不引入 HTML 显式锚点（`<a id>`），保持原生 Markdown。
- 不新建 references 文件（沿用路线 A）。
- 不改三个模板的章节骨架 / 数量 / 顺序（只在概览前新增目录段）。
