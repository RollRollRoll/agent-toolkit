# Codebase Analyzer

## 用途

对一个软件项目或源码仓库进行全面调研，输出一份结构化的中文调研报告，
让对项目不熟悉的人能够同时理解"它做什么"和"它怎么实现"。

支持三档输出深度：快速概览、标准调研（默认）、深度报告。

默认不读取真实 `.env`、私钥或疑似凭据文件；配置分析只使用示例配置和加载代码，报告不包含
环境变量真实值、token、密码、密钥或敏感默认值。

## 触发场景

- "调研这个项目 / 分析这个代码库 / 给我一份项目分析报告"
- "我要全面理解这个项目 / 帮我看看这个仓库 / onboarding 一下"
- 把一个不熟悉的代码目录交给 agent 并希望获得全局认识。
- 不适用：code review、定位 bug、评审具体实现、性能或安全诊断。

## 使用方式

将本目录下的 `SKILL.md` 和 `references/` 复制到目标平台的 skill 目录
（如 Claude Code 的 `.claude/skills/codebase-analyzer/`）即可直接使用。

## 目录说明

- `SKILL.md`：skill 主体（平台原生格式，含 frontmatter）。
- `references/`：调研清单、并行策略和三档输出模板。
- `docs/plans/`、`docs/specs/`：开发过程中的实现计划与设计文档。
